## 🚀 Featured Projects

### 🧠 Persistent Context Engine (PCE)
> *Operational memory for autonomous SRE — solving topology drift in distributed systems*

A high-scale memory substrate that preserves evolving system context across infrastructure mutations. When services rename, dependencies shift, and failure signatures morph, PCE keeps causal reasoning intact.

**The Hard Parts:**
- Built a **Topological-Invariant DAG (TI-DAG)** — all telemetry resolves to stable UUIDs, not service names, so infra mutations don't break historical causality
- Implemented **Bayesian confidence scoring** with entropy-based damping to prevent "noisy" central services from dominating causal narratives
- Engineered dual-mode reconstruction: **Fast Mode (p95 < 1.4s)** with LRU cache, **Deep Mode (p95 < 4.2s)** with full causal chain expansion
- Welford's online algorithm for rolling anomaly z-scores, Levenshtein fuzzy fingerprinting for behavioral matching across deployments

**Benchmark Results (Anvil P-02):**

| Metric | Target | Achieved |
|---|---|---|
| Incident Recall@5 | > 0.90 | **0.92** ✅ |
| Causal Accuracy | > 0.85 | **0.88** ✅ |
| Fast Mode p95 | < 2.0s | **1.4s** ✅ |
| Ingestion Throughput | > 1k ev/s | **1.8k ev/s** ✅ |

**Stack:** `Python 3.10+` · `Probabilistic Causal Graphs` · `BFS/DFS Traversal` · `Bayesian Inference` · `Docker`

### 🛡️ ShieldAI — Real-Time Content Moderation Platform
> *Multi-stage ML pipeline moderating millions of messages/day with sub-100ms latency on the fast path*

A production-grade content safety platform combining a lightweight ML classifier, LLM-based deep analysis, and a human reviewer console — all wired into an active learning feedback loop.

**Architecture Highlights:**
- **Two-stage ML cascade:** DistilBERT fast-path (< 50ms) → async LLM worker for borderline content, with conflict detection triggering active learning
- **Kafka-driven async pipeline** for text, image, and LLM workers — independently autoscaled by queue depth via Kubernetes
- **Configurable Policy Engine** backed by Redis — per-customer, per-region rules with zero DB hits per request
- **Human-in-the-loop feedback** — reviewer decisions flow back as labeled training data, prioritizing ML/LLM disagreements
- PII detection and redaction before content reaches external LLM APIs; mTLS between all microservices

**Stack:** `FastAPI` · `Python` · `Kafka/RabbitMQ` · `PostgreSQL` · `Redis` · `Docker + Kubernetes` · `HuggingFace Transformers` · `React` · `LangChain`

[![Repo](https://img.shields.io/badge/View_Repository-1a1a2e?style=for-the-badge&logo=github&logoColor=white)](https://github.com/VarunMA-23/Real-Time-Content-Moderation-Abuse-Detection-Platform)

### 📦 Stockly (ShelfIQ) — AI-Powered Retail ERP
> *A full SaaS ERP replacing 3–5 disconnected tools for small-to-medium retail businesses*

A cloud-based inventory management and POS platform with built-in ML dataset generation, real-time analytics, and AI forecasting architecture — purpose-built for Indian retail.

**What makes it real:**
- **Complete POS system** with JWT-auth, role-based access (Admin/Manager/Cashier/Warehouse), PDF invoice generation, and loyalty points
- **ML dataset pipeline:** every sale auto-generates a `DailyRecord` enriched with real-time weather (WeatherAPI), Indian festival detection (25+ festivals), seasonal context — exported as one-click CSV for model training
- **MongoDB aggregation pipelines** powering real-time KPIs, revenue charts, best-seller rankings, and peak-hours analysis
- Full purchase order lifecycle (draft → approval → received) with auto-refill recommendations based on stock thresholds
- Architecture documented across spec sheets, phased roadmaps, and agent definitions before a line was written

**Stack:** `React 18` · `TypeScript` · `Vite` · `Tailwind CSS` · `Node.js` · `Express` · `MongoDB` · `JWT` · `Recharts` · `shadcn/ui`

## 🤝 Engineering Mindset

I believe the best code is code that earns its complexity. Before building, I write specs. Before scaling, I understand the bottleneck. Before shipping, I think about what breaks.

A few things I hold to:

- **Ownership over credit** — if I said I'll do it, it gets done
- **Architecture first** — I document trade-offs before touching a keyboard
